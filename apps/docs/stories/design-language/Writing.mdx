# Writing Guidelines

UIテキストの書き方に関するガイドラインです。

---

## 基本原則

- **簡潔に** — 必要最小限の言葉で伝える
- **明確に** — 曖昧な表現を避け、具体的に書く
- **丁寧に** — 押しつけがましくない、穏やかなトーン

## マイクロコピー

### ボタン

- 動詞で始める: 「保存する」「削除する」「送信する」
- 具体的にする: 「送信する」＞「OK」
- 破壊的操作は明示する: 「アカウントを削除する」

### エラーメッセージ

- 何が起きたかを伝える
- どうすれば解決するかを示す
- 技術的な詳細は避ける

```
○ メールアドレスの形式が正しくありません
× Error: Invalid email format (RFC 5322)
```

### 空の状態

- 何もないことをポジティブに伝える
- 次のアクションを案内する

```
○ まだ投稿がありません。最初の投稿を作成しましょう。
× データがありません
```

### ローディング

- 長い処理は何をしているか伝える
- 短い処理はスピナーのみでよい

## 多言語対応

- テキストはハードコードせず、外部化できる構造にする
- 日本語と英語の両方で自然に読めることを確認する
- 日付・数値のフォーマットはロケールに依存する形で実装する
