# Terminology

nacalui で使う用語の定義です。プロジェクト全体で一貫した用語を使います。

---

## デザインシステム用語

| 用語 | 定義 |
| --- | --- |
| **Token（トークン）** | デザインの最小単位となる値。色、サイズ、余白など |
| **Palette（パレット）** | 生のカラー値のコレクション（stone-500, blue-600 など） |
| **Semantic Color（セマンティックカラー）** | 用途に紐づいた色の別名（fg-primary, bg-secondary など） |
| **Elevation（エレベーション）** | 要素の浮き上がり度合い。シャドウで表現する |
| **Spacing（スペーシング）** | 要素間の余白。4px基準のスケールで定義 |
| **Component（コンポーネント）** | 再利用可能なUIの構成要素 |
| **Variant（バリアント）** | コンポーネントの見た目や振る舞いのバリエーション |
| **Preset（プリセット）** | UnoCSS にトークンを適用する設定パッケージ |

## コンポーネント設計用語

| 用語 | 定義 |
| --- | --- |
| **Primary** | 最も重要なアクション・要素に使う |
| **Secondary** | 補助的なアクション・要素に使う |
| **Destructive** | 破壊的な操作（削除など）を示す |
| **Ghost** | 背景を持たない、最小限のスタイル |
| **Disabled** | 操作不可の状態 |

## サイズ命名

コンポーネントのサイズは以下の命名で統一します:

| 名前 | 用途 |
| --- | --- |
| **sm** | 省スペース、補助的な場面 |
| **md** | デフォルトサイズ |
| **lg** | 強調、タッチデバイス向け |
